Introduction

Purpose

The purpose of this document is to present a detailed description of the Online Recipe Book System. It will explain the purpose and features of the system, the interfaces of the system, what the system will do, the constraints under which it must operate and how the system will react to external stimuli. This document is intended for both the stakeholders and the developers of the system and will be proposed to the respective Faculty for its approval

Intended Audience and Reading Suggestions

This document is intended for both the stakeholders and the developers of the system and will be proposed to the respective Faculty for its approval.

Product Scope

This software will be an Online Recipe Book System for home cooks. This system will be designed to maximize the user’s interest by providing a huge number of recipes to choose from. It will offer easy and simple Instructions, along with specific ingredients for every recipe. Additionally, the Online Recipe Book will allow the user to easily generate a list of available ingredients from popular e-stores with lowest prices. They can then use this list to purchase from those specific stores manually. Moreover, the user will find suggestions of recipes based on popular ratings and nutrition value. The system also contains a relational database containing a list of Recipes, Ingredients, Instructions, Nutrition and Ratings.

Overall Description

Product Perspective

This System is a new, self-contained product. It consists of a website to view all viewable recipes. All recipes have relevant information about them and a functionality to generate a list depending on availability.

Product Functions

The Online Recipe Book has the following functionalities:

  • Browse Recipe
  • Search Recipe
  • Read Recipe Content
  • Generate Ingredients Price List
  • Recommend Recipe

User Classes and Characteristics

Admin Can Add, Edit and Delete Recipe entries
Reader Can read tutorials, read generated ingredient-list, view suggestions
System Can generate ingredient-list, generate tutorial suggestions

Operating Environment

  • Relational Database
  • Client/Server System
  • Operating System: Windows/Android
  • Database: SQLite and Firebase
  • Platform: Web and Android

Design and Implementation Constraints

SQL commands for queries/applications
Implement the database at least using a relational database management system

Assumptions and Dependencies

The factors being assumed that could affect the requirements are:

  • Availability of all possible ingredients in desired e-Stores.
  • Uptime of desired e-Stores
  • Availability of sufficient ratings for recommending recipes.

External Interface Requirements

User Interfaces

Front-End: JavaScript
Back-End: Django, SQLite, Java

Hardware Interfaces

Neither the mobile application nor the web version has any special hardware requirements. So, the system will not have any hardware interface.

Software Interfaces

Operating System We chose the Android operating system for its best support and user-friendliness.
Database To save recipe records, ingredients and instructions, we chose SQLite database
Framework To implement the project, we chose Django for its interactive support.

Communication Interfaces

This project supports all types of web browsers.

Other Nonfunctional Requirements

Performance Requirements

This section's requirements provide a detailed specification of the user interaction with the software and measurements placed on the system performance.
The performance requirements are:
● Ambiguous: The application will load within an adequate time frame.
● Unrealistic: The application will load in less than 0.01 seconds at all times.
● Unverifiable: Flexible, secure, sufficient, safe, ad hoc, adequate, user-friendly, usable, when required, if required, appropriate, fast, portable, lightweight, small, large, maximize, minimize, robust, quickly, easily

Safety Requirements

The requirements in this section provide a detailed specification of the user interaction with the software and measurements placed on the system performance. To ensure no data is lost in case the user decides to change devices or if the user's device is extensively damaged all data will be stored in cloud storages. This will allow safe storage and easy accessibility from anywhere.

Security Requirements

The System shall not disclose any personal information about the users. The Application shall not grant access to an unauthorized user and the Application shall not communicate with any other devices or servers while in use by the user. In addition to that users must refrain from uploading any sensitive information such as credit card info, id or any personal information that is not required.

Software Quality Attributes

● AVAILABILITY: The recipes will always be available on demand for users.
● CORRECTNESS: The recipes will be in correct form for respective food items.
● MAINTAINABILITY: The price of ingredients will be scraped periodically.
● USABILITY: The ease of our application should satisfy a maximum number of customers.

Business Rules

This application is free and we will not incorporate ads here.
